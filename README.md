# 餐厅小费数据分析项目

这是我做的第一个数据分析项目，同时用 **Python（Pandas）和SQL（SQLite）** 两种方式，分析了美国某餐厅的243条真实消费数据，探索影响顾客小费金额的关键因素，并输出可落地的餐厅运营建议。

---

## 用到的技术
- Python 3.12：作为数据分析的主语言
- Pandas：清洗数据、做统计分析和数据处理
- Matplotlib：绘制可视化图表（柱状图、折线图等）
- SQLite & SQL：使用SQL完成多维度分组统计、相关性计算等分析
- Seaborn：获取公开数据集

---

## 我发现的规律
- 晚餐比午餐贵，平均高3.6美元，小费也高0.4美元，晚上明显是餐厅最忙的时候
- 2个人一起吃饭的时候小费比例最高，能到16.6%；人越多，小费比例反而越低
- 男生平均花的钱比女生多一点，但女生给的小费比例稍微高一点
- 吸烟的人小费比例略高一点，但差异不大
- 周末生意最好，周日的平均账单是一周中最高的
- 账单越贵，小费越多，两者相关性很强
- 大部分人给的小费比例在13%-19%之间，平均是16.1%

---

## 给餐厅的建议
基于上面的发现，我觉得餐厅可以这么做：
- 晚餐和周末多安排几个服务员，不然忙不过来服务不好，小费会更少
- 可以多放一点2人桌，毕竟2个人来吃饭的最多，而且小费比例最高
- 对女生顾客可以稍微用心一点，她们给的小费更多
- 多人聚餐可以推出套餐，虽然他们小费比例低，但套餐能提高整体营收
- 周末可以搞点高端菜品或者特色活动，进一步提高客单价

## 项目文件
### Python版本
- `tips_analysis.py`：完整的Python数据分析代码
- `tips分析结果.csv`：清洗后的数据，包含计算好的小费率字段
- `时段账单对比.png`：午餐和晚餐的消费对比可视化
- `人数小费率关系.png`：用餐人数与小费比例的关系可视化

### SQL版本
- `create_db.py`：将原始数据集转换为SQLite数据库文件
- `sql_analysis.sql`：完整的SQL分析语句（多维度统计、相关性计算）
- `run_sql.py`：一键运行所有SQL查询，输出分析结果
- `tips.db`：生成的SQLite数据库文件

## 怎么运行
### Python版本
1. 安装依赖：`pip install pandas matplotlib seaborn`
2. 运行代码：`python tips_analysis.py`

### SQL版本
1. 运行`create_db.py`生成SQLite数据库文件`tips.db`
2. 运行`run_sql.py`一键执行所有SQL查询，查看分析结果
